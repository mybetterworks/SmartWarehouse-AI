## 2. 关键技术点

## 3. 开发经验

- 平台基座要同时交付后端和前端，否则登录、权限、菜单无法真实验证。
- Jenkins 应从基座版本开始接入，后续业务模块才能复用流水线。
- 正式环境发布基线越早建立，越容易暴露配置、镜像、密钥和健康检查问题。

## 4. 面试点

### Q1：为什么基座版本就接入 Jenkins 和正式环境？

答：这样后续每个模块都能沿用统一发布链路，避免最后集中补部署导致环境问题堆积。

### Q2：登录风控为什么需要 Redis？

答：登录失败次数和验证码状态需要在多实例之间共享，Redis 支持过期和原子计数，适合网关和 sys 多实例部署。
