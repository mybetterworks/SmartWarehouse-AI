## 2. 关键技术点

- MES 物料申请事件。
- WMS 库存预扣和分配结果事件。
- task 事件消费、统计、排行和预警联动。
- RabbitMQ eventId 幂等、重试、死信。
- 多乙方跨模块前后端联调。

## 3. 开发经验

- 多乙方交互应在各自模块可独立演示后再打通。
- 事件驱动能降低直接调用耦合，但必须做好幂等和补偿。
- 前端要展示跨模块状态变化，否则联动价值不直观。

## 4. 面试点

### Q1：MES 和 WMS 为什么适合事件驱动？

答：物料申请和库存分配存在异步处理、削峰和失败重试，用 RabbitMQ 事件可以降低耦合并提升容错。

### Q2：如何防止重复扣库存？

答：Redis Lua 保证预扣原子性，MQ 消费用 eventId 幂等，数据库用业务唯一键兜底。
