# 0003 敏捷版本拆分方案

## 状态

ACCEPTED

## 背景

项目需要通过 vibe coding 持续开发，并在每个版本后形成可演示或可运行成果。版本数量不能太多，以免管理复杂；也不能太少，否则每版改动过大，不利于学习、验证和复盘。

## 决策

初始版本拆分为 8 个 milestone：

| 版本 | 目标 |
|---|---|
| V01 | 项目骨架与本地基础设施 |
| V02 | 平台底座、网关与系统认证 |
| V03 | WMS 库存与离线上传 |
| V04 | MES 工单与物料申请 |
| V05 | 任务统计、排行、预警与 WebSocket |
| V06 | 前端门户与业务页面 |
| V07 | AI RAG、ChatBI、多 Agent 与 MCP |
| V08 | K8s、制品库、可观测性与整体联调 |

## 影响

1. `PROGRESS.md` 初始进度按 8 个版本管理。
2. `milestones`、`study`、`handle` 都按 8 个版本生成对应文件。
3. 每个版本完成后必须可演示或可运行。

## 后续动作

1. 如果某个版本过大，可以新增子版本或拆分新 milestone。
2. 如果版本拆分被调整，本文件改为 `SUPERSEDED`，并新增后续决策文件。
