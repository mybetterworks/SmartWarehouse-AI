# Study 目录

本目录用于记录每个版本完成后的技术总结、开发经验和面试点。

## 1. 使用规则

1. 每个 milestone 对应一个 study 文件。
2. 版本完成后必须更新对应 study。
3. study 重点服务于学习、复盘和面试表达，不替代正式设计文档。
4. 面试点要求问题有代表性，回答简洁、能体现项目经验。
