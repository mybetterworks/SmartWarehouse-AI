# Skill 模板

## 1. Skill 名称

```text
填写技能名称
```

## 2. 适用场景

说明什么时候使用这个 skill。

## 3. 输入

使用该 skill 前需要提供的信息：

- 相关设计文档。
- 当前版本 milestone。
- 目标服务或模块。
- 已有代码路径。
- 验收标准。

## 4. 执行步骤

1. 阅读上下文。
2. 检查当前实现。
3. 设计最小改动。
4. 实现功能。
5. 补充测试。
6. 运行验证。
7. 更新文档。

## 5. 检查清单

- [ ] 是否符合 `docs/design` 正式设计。
- [ ] 是否符合 `docs/plan/DEVELOPMENT_RULE.md`。
- [ ] 是否只修改当前版本范围。
- [ ] 是否补充必要测试。
- [ ] 是否更新 milestone、study、handle 和 PROGRESS。
- [ ] 如果该 skill 在使用中失效或被本次实践修正，是否已经同步更新 skill 内容。
- [ ] 是否检查并按需更新 `.gitignore`。
- [ ] 是否检查准备纳入 Git 的文件不存在隐私、敏感数据、账号、密码、token、API Key 或私钥。

## 6. 输出

使用该 skill 后应输出：

- 实现说明。
- 测试结果。
- 文档更新记录。
- 遗留问题。
- 如发生 skill 修正，输出失效原因、修正内容和后续适用范围。
- `.gitignore` 更新结果。
- 敏感信息检查结果。

## 7. 常见坑

记录容易遗漏的问题。

## 8. 失效修正规则

使用本 skill 时如果发现流程不再适用，应在完成当前问题修复后立即更新本文件：

1. 标明失效或不适用的步骤。
2. 写清修正后的操作流程。
3. 更新检查清单。
4. 补充新的常见坑。
5. 在当前版本 milestone、study 或 handle 中记录本次 skill 更新。
