## 2. 关键技术点

- AI MCP / REST 查询业务服务。
- ChatBI 查询库存、工单、排行和预警。
- 多 Agent 分析库存风险、登录风控和任务失败。
- Jenkins 全链路流水线。
- 阿里弹性容器正式部署、健康检查、回滚和可观测性。

## 3. 开发经验

- AI 接入业务数据后，必须强化只读权限、审计和超时控制。
- 正式发布不是最后才补，而是每个版本都积累发布能力，V08 只做交互加固和正式演示完善。
- 可观测性要覆盖跨服务 traceId，便于定位 AI 到业务服务的调用链。

## 4. 面试点

### Q1：AI 如何安全查询业务数据？

答：通过 MCP 或受控 REST 工具封装查询能力，ChatBI 使用只读账号、白名单表、SQL 类型限制、超时和最大返回行数。

### Q2：阿里弹性容器在项目中承担什么角色？

答：它承载正式环境部署和演示，使用正式镜像、正式配置、健康检查和回滚策略，区别于 Jenkins 的本地测试环境。
