## 3. 开发经验

- 工单状态机要先明确，避免页面和接口状态不一致。
- 物料申请先完成 MES 自身闭环，再进入跨 WMS 联动会更稳。
- 乙方前端只依赖平台 npm 包，不引用平台源码。

## 4. 面试点

### Q1：MES 物料申请如何保证幂等？

答：调用方传入 requestId，服务端对 requestId 建唯一约束，重复请求直接返回已有申请结果。

### Q2：为什么跨 WMS 联动不放在 MES 基础版本？

答：先让 MES 自身工单和申请流程稳定，再在后续交互版本打通 WMS，可以降低跨团队联调复杂度。
